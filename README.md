# CS300-Portfolio

Throughout this course, I developed an advising assistance program that uses data structures and algorithms to organize and search computer science course information. The main problem I solved was finding an efficient way to store course data so advisors could quickly display all courses in alphanumeric order and look up a specific course with its prerequisites. During the course, I implemented and analyzed three different data structures: a vector, a hash table, and a binary search tree (BST). By comparing these approaches, I learned how choosing the right data structure can greatly improve the performance and organization of a program.

At the beginning of the course, I focused mostly on getting a program to work correctly. As I progressed through the projects, I started thinking more about efficiency and scalability. The runtime analysis helped me understand why different data structures perform better for different operations. For example, vectors are simple to use but require sorting before displaying data and linear searches to find individual courses. Hash tables provide very fast lookups but do not automatically maintain data in sorted order. Binary search trees provide efficient searching while naturally storing data in sorted order, making them a good choice for this advising application. Learning these differences has given me a much better understanding of how to choose the appropriate data structure for a specific problem.

One of the biggest challenges I encountered was designing algorithms that correctly loaded course information from the input file while handling prerequisites and maintaining the proper relationships between courses. Another challenge was making sure the data structures behaved correctly when searching, inserting, and printing information. I overcame these obstacles by breaking the problem into smaller functions, testing each section independently, and carefully tracing through the program logic before combining everything together.

This project also changed the way I approach software development. Instead of immediately writing code, I now spend more time planning the solution before implementing it. Creating pseudocode and evaluating different algorithms made it much easier to organize the program and reduce errors during development. I also became more comfortable analyzing time complexity using Big O notation and considering both runtime performance and memory usage before selecting a data structure.

Finally, this course helped improve the quality of my programming practices. I now place greater importance on writing modular functions, using meaningful variable and function names, adding comments where appropriate, and organizing code so it is easier to read and maintain. These habits make programs easier to debug, modify, and expand in the future.
